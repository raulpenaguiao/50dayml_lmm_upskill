# Day 39: Building a Complete ML Pipeline - Further Reading

This day covers designing and implementing end-to-end ML pipelines from data processing through training to inference, essential for production systems.

## References

1. **Machine Learning Engineering for Production (MLOps)**
   - [Andrew Ng's MLOps Specialization](https://www.deeplearning.ai/program/machine-learning-engineering-for-production-mlops/)
   - Comprehensive course on designing production ML systems with pipelines and best practices.

2. **Orchestrating ML Workflows with Apache Airflow**
   - [Airflow Documentation](https://airflow.apache.org/)
   - Workflow orchestration platform for scheduling and monitoring complex pipelines.

3. **Kubeflow: Machine Learning on Kubernetes**
   - [Kubeflow](https://www.kubeflow.org/)
   - Kubernetes-native platform for ML workflows and deployment.

4. **Data Pipeline Best Practices**
   - [Fundamentals of Data Engineering](https://www.oreilly.com/library/view/fundamentals-of-data/9781098108298/)
   - Book covering data pipeline design, data quality, and batch vs streaming processing.

5. **Scikit-learn Pipelines**
   - [Scikit-learn Pipeline API](https://scikit-learn.org/stable/modules/compose.html)
   - Building reproducible preprocessing and modeling pipelines in Python.

---

**Tip:** Start with reference #5 for basic Python pipelines, #1 for complete systems, #2 for orchestration, and #4 for data engineering best practices.
